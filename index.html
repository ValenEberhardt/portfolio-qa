Estrategia de Pruebas: Aplicación Web Clima

En esta seccion voy a presentar una pequeña demostración de mis métodos a la hora de testear usando una aplicación de prueba creada para este propósito.

La aplicación a testear consume la API Open-Meteo lo que le permite mostrar datos meteorológicos de la ciudad que se inserte en el cuadro de búsqueda.

Cabe destacar que el análisis será abordado desde la perspectiva de un tester que tiene disponible los requerimiento funcionales de la aplicación, se aplicaron técnicas de black box testing para asegurar que la lógica y la interfaz responden según lo esperado

Criterio de Mejora: Algunos casos marcados como "Pasa" incluyen observaciones. Esto se debe a que, aunque cumplen con la función técnica, existen oportunidades para optimizar la UX y la eficiencia del flujo.

TC-01 Input en el buscador (Datos válidos) Falla Alta
  1. Ir a la página principal (URL: http://...)
  2. Ingresar una ciudad válida en la barra de búsqueda
  3. Clickear el botón buscar
  4. Esperar el tiempo correspondiente
  1. Se muestra la página principal con los siguientes elementos:
    • Texto de presentacion
    • Buscador de ciudades
    • Botón "Buscar"
  2. Una vez clickeado el botón "buscar" debe aparecer un mensaje de espera: "Cargando, por favor espere"
  3. El programa muestra el nombre, temperatura y humedad de la ciudad elegida o en su defecto un mensaje de error si la ciudad ingresada no se encuentra: "Ciudad no encontrada"

Ticket: BUG-01 - No se muestra la humedad

Severidad: Menor

Estado: Nuevo

Descripción: Al buscar una ciudad, el campo de humedad permanece vacío.

Prioridad: P2 (Mayor)

Evidencia: Ver Screenshot

Comentarios: A la hora de mostrar los resultados finales de la búsqueda, la sección de humedad permanece vacía mostrando solo un "%"..
TC-02 Input en el buscador (Datos inválidos) Pasa Alta
  1. Ir a la página principal (URL: http://...)
  2. Ingresar una ciudad inválida en la barra de búsqueda
  3. Clickear el botón buscar
  4. Esperar el tiempo correspondiente
  1. Se muestra la pagina principal con los siguientes elementos:
    • Texto de presentación
    • Buscador de ciudades
    • Botón "Buscar"
  2. Una vez clickeado el botón "buscar" debe aparecer un mensaje de error: "Ciudad no encontrada" / "Datos inválidos ingresados"

Comentarios:

Cuando se ingresa un dato inválido ("%, #, $") y se clickea buscar, el programa trata de buscar los caracteres inválidos en la base de datos de la API. ¿Es esto correcto? Personalmente opino que si se detectan caracteres inválidos antes de realizar la acción de Buscar, el proceso sería más eficiente.
